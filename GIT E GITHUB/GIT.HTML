VERSIONAMENTO DE CÓDIGO COM GIT E GITHUB

Então, quando fala em versionamento, estou falando em ter versões anteriores do meu projeto salvas para quando eu precisar alterar.

Desta forma, eu uso o GIT, que é um sistema de controle de versão, para controlar as versões do meu projeto. E o GITHUB é uma plataforma que me permite armazenar esses projetos na nuvem, além de me permitir compartilhar com outras pessoas.

Então, sempre que eu alterar algo no meu VSCode, ele irá aparecer no meu GIT, e quando eu quiser salvar essas alterações, eu faço um commit. E depois, quando eu quiser enviar essas alterações para o GITHUB, eu faço um push.

Lá no GITHUB eu terei acesso aos meus repositórios criados localmente após o processo acima descrito e poderei também, ver meu site no meu celular ou em outros computadores.

Passo a passo para hospedagem:

Crio o repositório tomando cuidado para o nome ser exatamente o mesmo da pasta, coloco o destino de onde ele será armazenado, coloco o README ativo. Dentro da pasta local eu renomeio o arquivo para index.html em minúsculo. Dou um push pra ele ir pro Github. Lá no github eu abro o settings do meu repositório, vou em pages no lado esquerdo, escolho a branch main e dou um save. Depois disso, eu copio o link que aparece e colo no meu navegador. E pronto, meu site está no ar.

Para dar manutenção no site, eu altero o que eu preciso no meu codigo, comito e pusho.

Se eu quiser trabalhar em outra maquina eu preciso ter o navegador, o vscode, git e github desktop. Assim eu logo no github, la no app logo no git tambem, crio uma pasta local de estudos, clono meu repositorio em que estava trabalhando para ela, trabalho no meu projeto, comito e pusho. E pronto, tudo atualizado.
