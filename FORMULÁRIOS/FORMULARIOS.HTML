Tudo sobre formulários

O Que São?

Formulários são uma parte essencial da web, permitindo que os usuários interajam com sites e aplicativos. Eles são usados para coletar informações, como nomes, endereços, emails e preferências.

Como Criar?

Eu quero criar, por exemplo, um nome e um sobrenome, com caixas de digitação ao lado e um botão de enviar e limpar embaixo.
Para isso eu devo usar a tag FORM e entro dela definir o que vai ser perguntado e o que vai ser enviado para o servidor.
Então eu crio um paragrafo ou uma div e escrevo o que eu quero que seja perguntado.
Para criar a "caixinha" ao lado do texto eu devo usar a tag INPUT e nela definir o tipo de INPUT que vai ser criado. Lista de Inputs mais adiante.

Elementos do Input

O input tem elementos como name e id que são usados para identificação.
Posso colocar names iguais para identificar um grupo de informações (Ex: pessoais, familia, contato, etc) e id que será relativo a cada input específico.
O name é o nome que vai ser enviado para o servidor e o id é a identificação do input.

Label

Quando eu crio um formulário, além de ter botões e caixinhas precisa haver semântica entre o input e o nome que está escrito ao lado.
Para isso eu devo usar a tag LABEL no texto fora da caixa (na pergunta) e dentro dessa tag colocar o id específico do input e não o name.
Desta forma, quando o usuário clicar no nome, a caixinha de texto vai ser selecionada pois existe uma correlação semântica entre a pergunta e a caixa de texto.

Métodos GET e POST

Existe também os métodos GET e POST que são usados para enviar os dados para o servidor.
O GET é usado para enviar dados sem segurança (ficam visíveis na url) e o POST é usado para enviar dados com mais segurança (ainda podem ser obtidos com o DevTools e por isso não é recomendado enviar dados sensíveis) e arquivos maiores (com mais de 3000 caracteres).

Atributos do Input

Atributos que posso definir no input são:

  • Placeholder: que é o texto que vai ser exibido dentro da caixinha de texto. Como exemplo: "Digite seu nome".
  • Required: que é um input obrigatório a ser preenchido. Se não for preenchido, o formulário não vai ser enviado e exibirá uma mensagem de erro automaticamente.
  • minlength e maxlength: que são o mínimo e máximo de caracteres que podem ser escritos na caixinha de texto.
  • size: que é o tamanho da caixinha de texto relacionada ao número de caracteres, lembrando que isso não é a quantidade de caracteres que pode ser escrito, mas sim o tamanho que será exibido.
  • autocomplete: que é um parâmetro utilizado no form (tag de abertura) e no input para definir se o navegador vai preencher automaticamente o input com os dados já escritos pelo usuário. Se eu habilitar isso no form (on) eu posso definir o tipo de autocomplete no input como senha, usuario, idade, email e etc.

Tipos de Input

Alguns formatos de input mais conhecidos são:

  • text - para criar um input de texto
  • password - para criar um input de senha
  • email - para criar um input de email
  • number - para criar um input de número
  • date - para criar um input de data
  • tel - para criar um input de telefone
  • color - para criar um input de cor (vai abrir uma paleta de cores em hexadecimal)
  • range - para criar um input de escolha (ex 1 a 5)
  • search - para criar um input de busca
  • url - para criar um input de url
  • week - para criar um input de semana
  • month - para criar um input de mês
  • checkbox - para criar opções de marcação de caixa
  • radio - para criar um grupo de inputs de mesmo nome
  • submit - para criar um botão de envio
  • reset - para criar um botão de limpar
  • button - para criar um botão
  • file - para criar um input de arquivo (escolher imagem por ex)
  • image - para criar um input de imagem
  • hidden - para criar um input oculto
  • datetime-local - para criar um input de data e hora
e em cada um deles posso posso definir o VALUE que é um valor padrão que vai ser exibido na caixinha de texto e que tenha relação com aquele dado pois será enviado posteriormente a um servidor (PHP) diferente do placeholder em que eu posso colocar o que eu quiser ou dar dicas de como preencher o input.

Expressões Regulares

Estudar mais sobre expressões regulares. REGEX. Pois posso utilizar elas para validar o input de email, telefone, etc com o atributo pattern.
Basicamente ele funcina com ! e o final da expressão $. Dentro disso eu defino as regras que o input deve seguir.

Fieldset e Legend

Posso ainda agrupar um conjunto de inputs em um grupo com a tag FIELDSET.
Dentro dela posso colocar um legend que é o título do grupo.
Essa tag de divisão é muito util para dividir o tipo de informação que eu quero coletar no meu formulário.
Exemplo: dados pessoiais, família, contato, etc.

Checkbox e Radio Button

Posso usar também checkbox para criar opções de marcação de caixa (posso marcar vários ou não marcar) e radio button para criar opções de marcação de botão (posso escolher um ou outro).
Importante lembrar que eu devo definir um VALUE para cada opção pois quando o usuário marcar a opção e enviar o formulário, o valor será enviado para o servidor sem o valor específico daquela opção e sim, on ou off somente.

Select

Posso usar também a tag select, que funciona igual tag INPUT, para criar uma lista de opções (ex escolha um estado).
Dentro dela posso usar a tag option para criar cada opção (nome de cada estado) e defino tb o VALUE para ser enviado ao servidor (tipo: MG, SP, RJ etc) e o texto que vai ser exibido na tela (ex Minas Gerais, São Paulo, Rio de Janeiro etc).
Posso também usar a tag optgroup para criar um grupo de opções (envelopo todas as optinos) e definir um label = nome do grupo.

Datalist

Posso usar também a tag datalist para criar uma lista de opções pre definidas com o diferencial de que o usuário pode digitar o que quiser e a tag vai sugerir uma opção.
Diferente da tag anterior que o usuário so pode escolher uma opção e não pode digitar.
Para isso abro a tag datalist e dentro dela crio uma tag option para cada opção.
Depois abro a tag input de texto e ao final dela defino o atributo list = nome da tag datalist que virá logo abaixo do input e dou um ID na datalist com o mesmo nome do input definido no list e pronto.
O usuário pode digitar o que quiser e a tag vai sugerir uma opção.
Posso também definir o VALUE de cada opção para ser enviado ao servidor.

Textarea

Posso usar também a tag textarea para criar um campo de digitação livre para o usuário escrever o que quiser.
Para isso abro a tag textarea, defino name e id, como normalmente faço no input e em seguida, para configurar o tamanho eu defino o atributo cols e rows.
Interessante usar o método post pois provavelmente será um arquivo com mais de 3kbytes e o método get não é recomendado para isso.
placeholder NÃO funciona aqui.

Output

Posso usar também a tag output para criar um campo de saída de dados.
Para isso abro a tag output, defino name e id, como normalmente faço no input
Posso, por exemplo, dizer que o output vai ser a soma de valores de outros campos (INPUTS) ou até mesmo o valor de um input de range que aparecerá ao lado da "barrinha" do range.
Então, dentro de cada input eu adiciono o atributo oninput = "o nome do id do OUTPUT.innerHTML (dizer que vai considerar o que esta escrito no html) = Number(id do input 1.value - valor digitado no campo do primeiro input) + Number(id do input 2.value - valor digitado no campo do segundo input)"
No caso do range, eu defino o atributo oninput = "o nome do id do OUTPUT.innerHTML (dizer que vai considerar o que esta escrito no html) = Number (id do input(range).value - valor digitado no campo do input(range)"
